# UI Audit - April 22, 2026

## Anti-Patterns Verdict

Fail, but not in a generic AI-dashboard way. The app has a distinct personality and avoids the usual purple gradients, glassmorphism, and card-grid sameness, but it still shows a few familiar AI-era tells: decorative motion stacked onto utilitarian flows, dense inline styling instead of a clearer system, and small "cute" controls that trade usability for charm. The main quality gap is not blandness, it is accessibility and resilience.

## Executive Summary

- Total issues found: 10
- Critical: 1
- High: 3
- Medium: 5
- Low: 2
- Top priorities:
  - Fix the verified production build failure in `HistoryScreen`
  - Harden the add-transaction dialog for keyboard and screen-reader use
  - Raise undersized touch targets across the app
  - Add persistent labels for placeholder-only inputs

## Detailed Findings

### Critical Issues

#### 1. Production build is currently broken

- Location: `app/components/HistoryScreen.tsx:64`, `tsconfig.json:3`
- Severity: Critical
- Category: Performance / Reliability
- Description: `next build` fails because `HistoryScreen` spreads `categoryCounts.entries()` while `tsconfig` targets `es5`.
- Impact: The app cannot produce a valid production build.
- Standard: Release-blocking reliability issue
- Recommendation: Replace the spread with `Array.from(categoryCounts.entries())` or raise the TypeScript target to ES2015+ if that matches browser support goals.
- Suggested command: `/harden`

### High-Severity Issues

#### 2. The add-transaction dialog is not keyboard-safe

- Location: `app/components/AddTransactionSheet.tsx:148`, `app/page.tsx:328`
- Severity: High
- Category: Accessibility
- Description: The sheet uses `role="dialog"` and `aria-modal`, but there is no focus trap, no `Escape` handling, no focus restoration, and outside-click close is mouse-only.
- Impact: Keyboard and screen-reader users can tab behind the modal and lose context.
- WCAG/Standard: WCAG 2.1.1 Keyboard, 2.4.3 Focus Order, 4.1.2 Name/Role/Value
- Recommendation: Trap focus while open, support `Escape`, return focus to the trigger on close, and make background content inert while the dialog is open.
- Suggested command: `/harden`

#### 3. Multiple controls miss the 44x44 touch-target minimum

- Location:
  - `app/components/AddTransactionSheet.tsx:58`
  - `app/components/AddTransactionSheet.tsx:733`
  - `app/components/MonthlyPlanningFlow.tsx:495`
  - `app/components/MonthlyPlanningFlow.tsx:591`
  - `app/components/HomeOverview.tsx:125`
  - `app/globals.css:566`
- Severity: High
- Category: Responsive / Accessibility
- Description: Several tappable controls are `28px` to `40px` tall.
- Impact: Mobile users get harder-to-hit targets and higher accidental-tap risk.
- WCAG/Standard: WCAG 2.5.5 Target Size (AAA), mobile ergonomics baseline
- Recommendation: Raise all primary and secondary interactive targets to at least `44px`.
- Suggested command: `/adapt`

#### 4. Several inputs rely on placeholder-only labeling

- Location:
  - `app/components/HomeScreen.tsx:135`
  - `app/components/AddTransactionSheet.tsx:304`
  - `app/components/AddTransactionSheet.tsx:576`
  - `app/components/AddTransactionSheet.tsx:708`
- Severity: High
- Category: Accessibility
- Description: Search, transaction-name, category-search, and date fields do not have persistent visible labels; some have no accessible label at all.
- Impact: Placeholder text disappears during input and weakens clarity for both screen-reader and sighted users.
- WCAG/Standard: WCAG 3.3.2 Labels or Instructions, 1.3.1 Info and Relationships
- Recommendation: Add explicit labels or `aria-label`s and keep contextual helper text visible where useful.
- Suggested command: `/clarify`

### Medium-Severity Issues

#### 5. Theming is split across duplicated font loading and mixed typography setup

- Location: `app/layout.tsx:25`, `app/globals.css:6`
- Severity: Medium
- Category: Theming / Performance
- Description: Fonts are loaded both through `<link>` tags and `@import`, and the declared families do not form a clean single system.
- Impact: Extra font requests and a muddier typography stack than necessary.
- Recommendation: Consolidate font loading to one approach and one intentional font pairing.
- Suggested command: `/normalize`

#### 6. Hard-coded colors bypass the token system

- Location:
  - `app/components/HomeScreen.tsx:171`
  - `app/components/HouseholdStatCard.tsx:122`
  - `app/components/HouseholdStatCard.tsx:257`
  - `app/components/MonthlyPlanningFlow.tsx:443`
- Severity: Medium
- Category: Theming
- Description: Status, partner, and button colors use raw hex values instead of reusable tokens.
- Impact: Theme changes become inconsistent and future visual cleanup gets more expensive.
- Recommendation: Move semantic colors into CSS variables and consume them consistently.
- Suggested command: `/normalize`

#### 7. Some text and state colors are likely contrast-risky

- Location:
  - `app/globals.css:27`
  - `app/components/HomeScreen.tsx:219`
  - `app/components/HistoryScreen.tsx:253`
  - `app/components/MonthlyPlanningFlow.tsx:575`
- Severity: Medium
- Category: Accessibility / Theming
- Description: `--muted` plus small mono text appears repeatedly on pale backgrounds.
- Impact: Low-vision users may struggle with metadata, helper copy, and warning text.
- WCAG/Standard: WCAG 1.4.3 Contrast (Minimum)
- Recommendation: Increase contrast for muted text or raise font size and weight where contrast must stay soft.
- Suggested command: `/quieter`

#### 8. Motion quality is uneven and sometimes overly decorative

- Location: `app/globals.css:1`, `app/components/HistoryScreen.tsx:313`
- Severity: Medium
- Category: Performance / Anti-Patterns
- Description: Some utility interactions use playful motion that feels ornamental rather than state-driven.
- Impact: The app can feel less intentional in high-frequency workflows.
- Recommendation: Reduce novelty in utility interactions and keep motion tied to feedback and state change.
- Suggested command: `/polish`

#### 9. The app shell is mobile-first to the point of being desktop-constrained

- Location:
  - `app/components/AppShell.tsx:34`
  - `app/components/BottomNav.tsx:31`
  - `app/components/MonthlyPlanningFlow.tsx:463`
- Severity: Medium
- Category: Responsive
- Description: Core layouts hard-cap around `480px` to `500px`, with limited adaptation for tablet and desktop.
- Impact: Larger screens feel like stretched phone shells instead of intentionally adapted layouts.
- Recommendation: Add breakpoint-specific layout behavior rather than relying on a narrow max-width wrapper.
- Suggested command: `/adapt`

### Low-Severity Issues

#### 10. Metadata and branding are inconsistent

- Location: `app/layout.tsx:5`
- Severity: Low
- Category: Theming / Polish
- Description: The title appears with broken emoji encoding and the app description is sparse.
- Impact: Browser tabs and PWA surfaces feel unfinished.
- Recommendation: Clean up title encoding and strengthen metadata copy.
- Suggested command: `/polish`

#### 11. There is dead-style gravity in `app/page.tsx`

- Location: `app/page.tsx:907`
- Severity: Low
- Category: Performance / Maintainability
- Description: Several style constants appear unused by the rendered tree.
- Impact: This increases cognitive load and makes future maintenance harder.
- Recommendation: Remove or relocate unused presentation constants.
- Suggested command: `/distill`

## Patterns And Systemic Issues

- Inline styles dominate the app, which makes consistency and responsive refinement harder.
- Small controls recur across screens, so ergonomics is a systemic problem rather than a one-off.
- Accessibility semantics are partially present, but dialog and form behaviors are incomplete.
- The UI has personality, but delight sometimes overrides utility in dense budget workflows.

## Positive Findings

- The app has a more distinctive visual voice than most CRUD finance tools.
- Reduced-motion handling exists in global CSS and is applied in several animated areas.
- The home summary card feels more bespoke than generic dashboard filler.
- Empty states and microcopy feel human rather than boilerplate.
